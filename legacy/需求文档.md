# RSSHub + JAVBus 演员订阅功能需求文档

## 1. 项目概述
项目名称: JAVBus 演员 RSS 订阅系统
功能概述: 通过 Docker 部署 RSSHub，抓取 JAVBus 演员页面更新，自动推送新影片信息
目标用户: 影视爱好者、影片收藏者
---

## 2. 功能需求

### 2.2 演员数据管理
[ ] 对现有产品的演员页面进行更新
[ ] 为每个演员记录 JAVBus 编号（ actresses ID）
[ ] 建立演员本地数据库（演员名 ↔ JAVBus ID 映射）
### 2.3 RSS 订阅生成
[ ] 为每个演员生成对应的 RSS 订阅链接
[ ] 订阅链接格式: `RSSHub地址/javbus/actor/{actor_id}`
[ ] 支持批量生成和管理订阅列表
### 2.4 自动更新机制
[ ] 软件启动时自动更新所有演员的 RSS 订阅
[ ] 对比历史记录，筛选出新发布的影片
[ ] 展示新影片列表（标题 + JAVBus 地址）
### 2.5 已阅功能
[ ] 支持对列表一键标记已阅
[ ] 记录已阅状态，避免重复展示
### 2.6 RSS 展示界面
[ ] 总列表视图: 所有演员的新影片汇总
[ ] 按演员视图: 按演员分组展示各自的新影片
[ ] 支持切换两种视图
---

## 3. 验收标准
[ ] 演员 JAVBus 编号正确关联
[ ] 启动软件能自动抓取并展示新影片
[ ] 一键已阅功能正常
[ ] 总列表和按演员列表两种视图可用
[ ] 影片权重打分正确计算
---

## 5. 优先级
P0: 演员 JAVBus ID 关联
P1: RSS 订阅生成 + 启动自动抓取
P2: 展示界面（总列表/按演员）
P3: 一键已阅功能
---

## 6. 影片权重打分机制

| 条件 | 加分 |
|------|------|
| 初始分 | 50 分 |
| 同时上榜 JAVDB + JAVLibrary | +30 分 |
| 单独上榜 JAVDB/JAVLibrary | +20 分 |
| 仅上榜 JAVDB 年度榜单 | +10 分 |
| 多位演员（≥2人） | +5 分 |
| 演员在关注榜单（RSS标签） | +15 分 |

# 需求更新

1. 我希望从演员列表点进去之后的演员能够有一个演员的“主页”，

   主页上可以有当前演员的头像照片（可以通过jellyfin服务器获取）
   在这个主页上维护“javbus”的ID。并且可以增加一个最新RSS更新的日期、当前影片数量
   演员主页可以分为2个Tab页，来展示所有的影片和RSS订阅列表。
   不要在演员列表去维护Javbus的ID

2. 权重打分不应该是在RSS订阅列表的，应该是在全部影片列表、演员影片列表中。应该是在目前的每个影片的海报页面的下面有一个比较显眼的地方有他的权重分。同时所有的影片列表都应该支持按照权重分进行排序

3. RSS订阅还应该增加一个功能，即和当前演员已经有的影片进行比对。如果该影片已经出现在jellyfin中了，那么应该自动勾上已经订阅。（主要在首次订阅时会出现影片数量巨大的问题）
